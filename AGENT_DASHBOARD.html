AutoMind Agent 🧠

Central Dashboard & Documentation for the autonomous DevOps AI agent.

🤖 What the Agent Does

AutoMind is a Markdown-based autonomous DevOps AI agent. Its primary responsibilities are:

  • Repository Monitoring: Continuously watches configured local Git repositories for new commits and changes.
  • Jira Correlation: Automatically parses commit messages and branch names to link code changes to Jira tickets using AI and regex (e.g., SCRUM-123).
  • Smart Status Updates: Analyzes the implementation completeness from diffs and moves Jira tickets to appropriate statuses such as "In Progress" or "Human-Review" / "Move to QA".
  • Team Communications: Generates professional, client-ready summaries of code changes and broadcasts them via Slack Webhooks.
  • Self-Healing & Resilience: Automatically handles API rate limits (e.g., HTTP 429), retries failed requests, and manages long-running queues without hanging.

🚀 How to Run It

1. Environment Variables

Create a .env file in the root directory:

JIRA_API_TOKEN=your_jira_token
SLACK_WEBHOOK_URL=your_slack_webhook_url

2. Global Configuration

Update memory/config.md with your Jira email, Jira Base URL, and monitoring preferences.

3. Add Repositories

Create a markdown file in memory/repos/ (e.g., my-app.md): 

# My App Repository
- REPO_PATH: /path/to/your/local/repo
- REPO_BRANCH: main

4. Launch the Agent

claude --dangerously-skip-permissions

✅ Guidelines & Checklists

Pre-Run Checklist

Operational Guidelines

  • Commit Best Practices: Always include the Jira Ticket ID (e.g., PROJ-404) in your commit messages or branch names.
  • State Management: The agent maintains its working state in memory/state.md. Do not edit this file manually while the agent is actively running.
  • Troubleshooting: Check the troubleshooting-[timestamp].md files in the memory/ directory if unrecoverable errors occur.
  • Rate Limits: Do not force-restart repeatedly if Jira returns HTTP 429; allow the agent's built-in timeout to handle it.
